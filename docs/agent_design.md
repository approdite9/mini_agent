# Agent 设计要点

一个最小可用的 Agent 至少包含四个部分：

1. **循环思考（ReAct Loop）**：接收用户输入 → 模型判断直接回复还是调用工具 →
   执行工具 → 把结果作为观察反馈给模型 → 直到模型给出最终答案或达到轮次上限。
2. **工具与注册机制**：每个工具声明名称、描述、参数 JSON Schema，
   模型依据 schema 自主决策调用哪个工具、传什么参数。
3. **会话与记忆**：多窗口独立对话历史；记忆可全局共享，也可按会话隔离。
4. **可观测性**：思考过程、工具调用、工具结果、错误全部记入 trace 日志。

## 错误处理原则

- 工具执行失败不应终止循环，而应把错误信息反馈给模型，让其重试或改道。
- LLM 输出不符合协议时，把格式错误回传给模型自我纠正。
- 设置最大轮次上限，防止死循环。
